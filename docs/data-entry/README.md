# Pertinence Dashboard — Data Entry Standards

This folder is the handover guide for the Google Sheets that feed the Pertinence Dashboard. Each team has its own short document. **Everyone** reads the Common Rules first.

## Read first

- **[00-common.md](00-common.md) — Common Rules.** How the dashboard reads sheets, universal rules that apply to every sheet, the process for adding new locations / purposes / categories, **the year-rollover process for 2027 and beyond ([§6](00-common.md#6-starting-a-new-year-2027-and-beyond))**, troubleshooting, and the glossary. ~10 minutes; read once.

## How to use these documents

1. Read **00-common.md** once. Keep it findable — it's the rule book.
2. Read your team's document in full. Refer back when you're entering new types of data or when something looks wrong on the dashboard.
3. When in doubt, **do not "fix" a value to look right on the dashboard**. The dashboard's Data Quality view exists to surface unusual values for the supervisor to decide on. Editing a sheet to match the dashboard hides real problems.
4. If a sheet column, header, or tab needs to change, contact the dashboard owner **before** changing it — even a renamed header can hide a whole column from the dashboard.
